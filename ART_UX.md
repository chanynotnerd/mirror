# Mirror *(가제)* — 아트 & UX 가이드

> 비주얼·UI·손맛을 정의하는 문서. 게임 규칙은 `GDD.md`, 기술은 `SPEC.md` 참조.
> "깔끔한 완성도"가 이 게임의 핵심 무기 — 이 문서가 곧 차별화 포인트.

---

## 1. 아트 방향성
- **미니멀 + 네온.** 차분한 다크 배경에 채도 높은 네온 2색이 튀는 대비.
- **도형 기반.** 캐릭터=원, 장애물=라운드 사각, 코인=작은 원/다이아. 일러스트 에셋 없음.
- **원칙:** 에셋 제작을 최소화해서 절약한 자원을 *손맛 연출*에 전부 투입한다.
- 다크 우선. UI 전체가 다크 톤.

## 2. 컬러 토큰

| 역할 | Hex | 비고 |
|------|-----|------|
| 배경 (base) | `#0E1117` | 가장 어두운 바탕 |
| 표면 (surface) | `#161C24` | 패널·버튼 바탕 |
| 미러 라인 | `#2A313B` | 중앙 분할선 (은은하게) |
| 플레이어 (기본) | `#34E1C4` | 네온 시안 — 스킨으로 변경 |
| 코인 | `#FFC24B` | 네온 앰버 |
| 콤보/강조 | `#FF5C8A` | 네온 핑크 |
| 위험/충돌 | `#FF4D4D` | 사망·경고 |
| 텍스트 (주) | `#F4F4EF` | |
| 텍스트 (보조) | `#93958F` | |

스킨은 **플레이어 색만** 교체: `default #34E1C4` / `neon_a #3D8BFF` / `neon_b #FF5C8A`.

## 3. 타이포그래피
- **폰트 1종**, 지오메트릭 산세리프. 한글 포함이면 **Pretendard** 추천(영문·숫자 깔끔, 무료).
- 점수는 **탭룰러 숫자**(자릿수 흔들림 방지), 굵게.
- 위계: 점수 표시 40pt+ / 화면 타이틀 28pt / 버튼 18pt / 보조 14pt.
- 전부 문장형, 대문자 떡칠 금지.

## 4. 셰이프 언어
- 캐릭터: 정원, 약간의 글로우(은은하게).
- 장애물: `border-radius` 큰 라운드 사각, 무채색에 가까운 표면색.
- 코인: 작은 원 + 회전 반짝임.
- UI: 라운드 코너(12–16px), 얇은 테두리.

## 6. HUD 규칙
- 점수: 상단 중앙, 가장 크게. 코인: 상단 우측. 일시정지: 상단 좌측.
- 게임 플레이 영역(두 레인)은 절대 가리지 않는다.
- 콤보 시 점수 옆에 배수(`x2`)가 핑크로 팝.

## 7. 손맛 & 피드백 스펙 — **최우선**

| 이벤트 | 햅틱 | 사운드 | 비주얼 | 시간 |
|--------|------|--------|--------|------|
| 갭 통과 | Light impact | 짧은 "틱" (콤보마다 반음↑) | 플레이어 살짝 스쿼시 | ~80ms |
| 퍼펙트 통과 | Medium impact | 밝은 틱 + 반짝 | 링 펄스 + 콤보 배수 팝 | ~200ms |
| 코인 획득 | Selection | 코인 차임 | 스파클 6–8개 + 코인이 HUD로 날아감 | ~300ms |
| 충돌(사망) | Heavy + notification | 임팩트 thud | 화면 흔들림 + 양쪽 파티클 폭발 + 0.2초 슬로우모션 + 적색 플래시 | ~250ms |
| 버튼 탭 | Selection | UI 클릭 | 스케일 0.96 눌림 | ~120ms |

세부 수치:
- 화면 흔들림: 진폭 12px, 250ms에 걸쳐 감쇠.
- 슬로우모션: timeScale 0.3, 200ms 후 복귀.
- 사망 파티클: 레인당 ~20개, 플레이어 색.
- 콤보 음정: 5음계(펜타토닉)로 한 단계씩 상승 → 거슬리지 않게.

## 8. 모션 원칙
- 트랜지션 150–250ms, ease-out 기본.
- 화면 전환·재시작 페이드 300ms.
- **`prefers-reduced-motion` 존중** — 흔들림·슬로우모션 약화 옵션 제공.

## 9. 사운드 디자인
- 최소 팔레트: 통과 틱 / 코인 차임 / 사망 thud / UI 클릭. BGM은 없거나 아주 은은하게.
- 콤보는 펜타토닉 음정 상승으로 *성취감*을 청각화.
- **저작권 주의:** CC0 또는 자체 제작 음원만 사용 (출시 필수 조건).

## 10. 접근성
- **색에만 의존 금지** — 플레이어와 장애물은 *형태*로도 구분(원 vs 사각).
- 사운드·햅틱 개별 on/off.
- 색약 대비: 플레이어/코인/위험 색이 명도로도 구분되게.
